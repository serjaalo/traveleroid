## 1.6 Хостинг веб-приложений на платформе SprintBox

Для того чтобы разработанное веб-приложение было доступно конечным пользователям, его необходимо разместить на специализированной серверной площадке — хостинге. Хостинг предоставляет вычислительные ресурсы, постоянное сетевое подключение, доменное имя и защищённое соединение по протоколу HTTPS.

В качестве хостинговой платформы для проекта выбран сервис **SprintBox**. Это PaaS-платформа (Platform as a Service), ориентированная на размещение современных веб-приложений на Node.js, в том числе построенных с использованием фреймворка Nuxt. В отличие от классических VPS-серверов, на которых разработчику приходится самостоятельно настраивать операционную систему, веб-сервер, процессы и SSL-сертификаты, SprintBox автоматизирует эти задачи и предоставляет готовую среду исполнения.

Ключевые возможности платформы SprintBox, существенные для данного проекта:

- интеграция с Git-репозиторием и автоматическое развёртывание (continuous deployment) при каждом коммите в основную ветку;
- автоматическая сборка проекта по командам, описанным в `package.json`;
- безопасное хранение переменных окружения (секретов сессии, паролей администратора);
- автоматическая выдача и продление SSL-сертификатов, поддержка HTTPS «из коробки»;
- предоставление публичного домена и возможность подключения собственного;
- поддержка постоянных WebSocket-соединений, что необходимо для реализованного в приложении модуля чата на базе библиотеки socket.io;
- встроенные журналы (logs) и метрики использования ресурсов;
- возможность горизонтального масштабирования при росте нагрузки.

Использование SprintBox позволяет сосредоточить усилия на разработке функциональности приложения и снизить трудозатраты на администрирование серверной инфраструктуры, что особенно важно в рамках выпускной квалификационной работы.

## 2.5 Развёртывание веб-приложения на хостинге SprintBox

Для размещения разработанного веб-приложения на платформе SprintBox были выполнены следующие шаги.

1. **Подготовка проекта.** В файле `package.json` определены команды сборки (`npm run build`, выполняющая `nuxt build`) и запуска production-сервера (`node .output/server/index.mjs`). Все секреты вынесены в переменные окружения; шаблон значений хранится в файле `.env.example`. Локально проверена корректность сборки и запуска.
2. **Создание проекта в SprintBox.** В личном кабинете платформы создан новый проект типа *Node.js application* с именем `travelapp`, которому присвоен публичный адрес вида `https://travelapp.sprintbox.app`.
3. **Привязка Git-репозитория.** В качестве источника кода подключён GitHub-репозиторий проекта, выбрана основная ветка `main` для автоматического развёртывания.
4. **Настройка сборки и запуска.** В разделе *Build & Run* указаны: команда установки зависимостей `npm ci`, команда сборки `npm run build`, команда запуска `node .output/server/index.mjs`, версия Node.js — 20.x (LTS), порт приложения — 3000.
5. **Переменные окружения.** В защищённом разделе *Environment Variables* добавлены значения `NUXT_SESSION_PASSWORD` (секрет для подписи сессий пользователей), `ADMIN_PASSWORD` (пароль доступа к админ-панели) и `NODE_ENV=production`.
6. **Домен и HTTPS.** Платформа автоматически выпустила SSL-сертификат и сделала приложение доступным по HTTPS на выданном поддомене.
7. **Хранение пользовательских файлов.** Загружаемые пользователями фотографии (для постов, аватаров и сообщений в чате) сохраняются в каталог `public/uploads`, смонтированный к постоянному файловому хранилищу SprintBox (persistent volume), что обеспечивает сохранность данных при перезапусках.
8. **Поддержка чата.** Поскольку в приложении используется socket.io, для модуля чата требуются постоянные WebSocket-соединения. SprintBox поддерживает их без дополнительной настройки, что подтверждено успешной отправкой и приёмом сообщений в реальном времени между двумя пользователями.
9. **Автоматический деплой.** Каждый коммит в ветку `main` инициирует новую сборку и развёртывание; при возникновении ошибок предыдущая рабочая версия остаётся доступной (zero-downtime deployment). Прогресс отслеживается в журнале раздела *Deployments*.
10. **Мониторинг.** В интерфейсе платформы доступны журналы stdout/stderr приложения, графики использования CPU и оперативной памяти, а также текущий статус запущенных инстансов.

По завершении настройки проведена итоговая проверка: пройдены сценарии регистрации пользователя, создания поста с фотографией, обмена сообщениями в чате и работы административной панели. Все функции веб-приложения корректно работают на удалённом сервере, что подтверждает успешное развёртывание системы.

## Заключение по разделам 1.6 и 2.5

В теоретической части (п. 1.6) обоснован выбор PaaS-платформы SprintBox в качестве хостинга для разработанного веб-приложения: она удовлетворяет всем функциональным требованиям проекта (поддержка Node.js/Nuxt, WebSocket для чата, HTTPS, переменные окружения, постоянное хранилище) и снижает накладные расходы на администрирование инфраструктуры.

В практической части (п. 2.5) выполнено пошаговое развёртывание приложения **TravelApp** на SprintBox: настроены автоматическая сборка из Git-репозитория, защищённое HTTPS-соединение, переменные окружения, постоянное хранилище для пользовательских файлов и поддержка WebSocket-соединений модуля чата. В результате веб-приложение доступно по публичному адресу и готово к демонстрации.
